2) BANKROLL + DYNAMIC BET SIZING
Add bankroll and betting to the state and settlement logic.
- On starting session, set bankroll to starting value.
- Before each new hand is dealt, compute recommended bet using true count:
  - Use a simple ramp (units):
    - TC <= 0 => 1 unit
    - TC 1 => 2 units
    - TC 2 => 4 units
    - TC 3+ => 6 units
  - Cap units at max_units from settings (default 8)
  - bet = min_bet * units
  - bet must not exceed bankroll (if bankroll smaller, bet = bankroll or minimum allowed, but be consistent and document)
- Splits:
  - Splitting requires placing an additional bet equal to the original bet for the new hand (deduct from bankroll at split time).
- Doubles:
  - Doubling requires placing an additional bet equal to the hand’s current stake (deduct at double time).
- Settlement:
  - For each hand, apply outcome using that hand’s stake:
    - win: bankroll += stake
    - push: bankroll += 0
    - lose: bankroll -= 0 (if you already deducted stake up front) OR bankroll -= stake (if not deducted up front)
  Choose ONE consistent accounting model:
    Preferred: “stake-at-risk” model:
      - Deduct stake from bankroll when bet is placed
      - On win: bankroll += 2*stake (return stake + profit)
      - On push: bankroll += stake (return stake)
      - On loss: bankroll += 0
  Implement correctly for split and double.
- Display bankroll + current bet prominently.
